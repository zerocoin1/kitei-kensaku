# 管理者マニュアル（非エンジニア向け）

このマニュアルどおりに進めれば、専門知識がなくても導入・運用できます。
所要時間はおよそ **1〜1.5時間**（初回のみ）。以降の運用は「PDF上書き＋ボタン1回」だけです。

---

## 全体像（何をするのか）
```
[管理者] Google Driveのフォルダに最新PDFを上書き
              ↓（アプリが自動でダウンロード・ベクトル化）
[アプリ] 会社・種目で絞ってGeminiが回答＋根拠ページを表示
              ↓
[50人のメンバー] ブラウザから共通パスワードで利用
```

準備するものは3つの「鍵」です。
1. **Gemini APIキー**（AIの利用鍵。実費課金）
2. **サービスアカウント**（アプリがDriveを読むためのロボット用アカウント）
3. **共通パスワード**（メンバーがアプリに入るための合言葉。あなたが自由に決める）

---

## ステップ1：Gemini APIキーを取得する
1. ブラウザで **Google AI Studio**（aistudio.google.com）を開き、Googleアカウントでログイン
2. 左メニューの「**Get API key（APIキーを取得）**」→「**Create API key**」
3. 表示された `AIza...` で始まる文字列をコピーして、メモ帳に一時保存
   - これが `GOOGLE_API_KEY` になります
4. 課金設定：パイロット規模（50人・月数百〜数千円）なら無料枠〜低額です。請求先を設定しておくと安定します

---

## ステップ2：Google Drive連携（サービスアカウント）を作る
アプリはあなた個人としてではなく「ロボット用アカウント（サービスアカウント）」として
Driveを読みます。閲覧専用なので安全です。

### 2-1. プロジェクトとAPIを有効化
1. **Google Cloud Console**（console.cloud.google.com）を開く
2. 画面上部でプロジェクトを新規作成（名前は「rag-hoken」等なんでもよい）
3. 検索窓に「**Google Drive API**」と入力 → 開いて「**有効にする**」

### 2-2. サービスアカウントを作成
1. 左メニュー「**IAMと管理**」→「**サービスアカウント**」→「**サービスアカウントを作成**」
2. 名前を入力（例：`rag-reader`）→「作成して続行」→ 役割は不要なので「完了」
3. 作成された行のメールアドレス（例：`rag-reader@xxx.iam.gserviceaccount.com`）を**コピー**
   - これが後で「Driveフォルダを共有する相手」になります

### 2-3. 鍵（JSON）をダウンロード
1. そのサービスアカウントをクリック →「**キー**」タブ →「**鍵を追加**」→「**新しい鍵を作成**」
2. 形式は **JSON** を選び「作成」→ JSONファイルが自動ダウンロードされます
3. このファイルは**厳重に保管**（他人に渡さない。GitHubにも上げない）

### 2-4. Driveフォルダを準備して共有
1. Google Driveで規定類の親フォルダを作る。中は必ず **会社→種目** の2階層に：
   ```
   保険規定データ/           ← これが「親フォルダ」
     ├─ A損害保険/
     │    ├─ 自動車保険/
     │    │    └─ 約款2026.pdf
     │    └─ 火災保険/
     │         └─ ハンドブック.pdf
     └─ B損害保険/ ...
   ```
   - フォルダ名は `rag/config.py` の `COMPANIES` / `PRODUCT_TYPES` と**完全一致**させる
2. 親フォルダを右クリック →「**共有**」→ ステップ2-2でコピーした
   サービスアカウントのメールを追加し、権限は「**閲覧者**」でOK →「送信」
3. 親フォルダを開き、URLの末尾の文字列（`folders/` の後ろ）をコピー
   - 例：`.../folders/1AbCdEfGhIJ...` の `1AbCdEfGhIJ...` が `drive_root_folder_id`

> 💡 補足：うまくいかない時は「共有ドライブ（Shared Drive）」に親フォルダを置き、
> そこにサービスアカウントを追加すると安定します。

---

## ステップ3：ローカルPCで動作確認（任意だが推奨）
1. アプリ一式のフォルダを開き、`.streamlit/secrets.toml.example` を
   コピーして `.streamlit/secrets.toml` にリネーム
2. `secrets.toml` を開き、次を記入：
   - `GOOGLE_API_KEY` … ステップ1のキー
   - `app_password` … 好きな合言葉
   - `drive_root_folder_id` … ステップ2-4のID
   - `[gcp_service_account]` … ステップ2-3のJSONファイルの中身を各項目に転記
     （`private_key` は改行が `\n` のままでOK）
3. コマンドプロンプトで以下を実行：
   ```
   pip install -r requirements.txt
   streamlit run app.py
   ```
4. ブラウザで開いたら合言葉を入力 →「🔄 データ更新」を押す
   → PDFが取り込まれ、検索できれば成功

---

## ステップ4：Streamlit Community Cloudへ公開（50人が使える状態に）
1. **GitHub**（github.com）でアカウントを作り、**非公開（Private）リポジトリ**を作成
2. アプリ一式をアップロード
   - ⚠️ `secrets.toml` と ダウンロードしたJSON鍵は**絶対にアップしない**
     （`.gitignore` で除外済みですが念のため確認）
3. **share.streamlit.io** にGitHubでログイン →「**New app**」→ 該当リポジトリと `app.py` を選択
4. 「**Advanced settings → Secrets**」に、ローカルの `secrets.toml` の中身を**丸ごと貼り付け**
5. 「Deploy」→ 数分でURLが発行される。そのURLと共通パスワードを50人に共有

> ⚠️ **注意**：Community Cloudは一定時間アクセスがないとスリープし、
> 復帰時にインデックスの再構築（1〜2分）が走ります。最初のアクセス者が
> 「🔄 データ更新」を押せば復旧します。常に即応させたい場合は
> Google Cloud Run など常時稼働ホストへの移行を検討してください。

## 日々の運用（ここだけ覚えればOK）
### 規定が改定されたら
1. Google Driveの該当フォルダに**新しいPDFを上書きアップロード**
   （ファイル名は同じでも別名でもOK）
2. アプリを開き「🔄 データ更新」を**1回押す**
3. 完了メッセージが出れば、全員が最新データで検索できます

> これで「各自がPDFを読み込ませ直す手間」と「古いデータでの誤回答（先祖返り）」が
> 撲滅されます。

---

## 困ったときは（トラブル対処）
| 症状 | 原因・対処 |
|---|---|
| 「対象PDFが見つかりません」 | フォルダ構造が `会社/種目/xxx.pdf` の2階層になっているか、フォルダ名がconfigと一致しているか確認 |
| 「テキストを抽出できませんでした」 | スキャン（画像）PDFの可能性。文字が選択できるPDFを用意（OCRは将来対応） |
| Drive同期でエラー | サービスアカウントに親フォルダを「閲覧者」で共有したか、Drive APIを有効化したか確認 |
| いつも「記載がありません」になる | 選んだ会社・種目が正しいか、`rag/config.py` の `SIMILARITY_CUTOFF` を少し下げる（例:0.25） |
| 回答が遅い/エラー | APIキーの課金設定、または一時的なAPI混雑。少し待って再実行 |

## 設定を変えたいとき（`rag/config.py`）
- **会社・種目の追加/変更**：`COMPANIES` / `PRODUCT_TYPES` を編集（Driveのフォルダ名と一致させる）
- **モデルを新しくする**：`LLM_MODEL`（例 `gemini-2.5-flash`）を差し替え
- **検索の広さ/厳しさ**：`TOP_K`（拾う件数）、`SIMILARITY_CUTOFF`（関連度の足切り）を調整
- **Drive不要のローカル運用**：`SOURCE_MODE = "local"` にして
  「Googleドライブ デスクトップ」で同期したフォルダを `data/pdfs/` に向ける運用も可能
